🔬 Detailed Workflow Steps

⚙️ train_model.py — What It Does

  • 1
    Loads training.csv (440K rows × 12 cols)
  • 2
    Drops CustomerID — identified as data leakage in your EDA
  • 3
    Removes rows with NaN values to ensure clean input
  • 4
    Encodes categorical columns: Gender, Subscription Type, Contract Length using LabelEncoder
  • 5
    Splits data 80/20 with stratification to preserve churn ratio
  • 6
    Trains RandomForestClassifier (100 trees, balanced class weights)
  • 7
    Prints AUC, Classification Report, and Feature Importances
  • 8
    Saves model + 4 encoders as .joblib files

🖥️ churn_app.py — UI Tabs

  • 🎯
    Individual Prediction — Sliders for age, tenure, support calls, payment delay, contract type, spend, usage. Returns a gauge chart + risk badge + personalised insights.
  • 📊
    Risk Analysis — Feature importance bar chart, risk tier definitions table, churn rate by contract type chart.
  • 📂
    Batch Processing — Upload any CSV matching the column schema, get predictions for all rows, download results sorted by churn risk.
  • 📖
    How It Works — Full pipeline diagram, file structure tree, key EDA findings, and this guide embedded in the app.
📋 Input Features & Churn Impact
Feature Type Range / Values Churn Impact Notes from EDA
Support Calls Numeric 0 – 10 🔴 Very High Correlation: 0.51 with churn — strongest predictor
Contract Length Categorical Annual / Quarterly / Monthly 🔴 Very High Monthly: ~62% churn; Annual: ~10% churn
Payment Delay Numeric 0 – 30 days 🔴 High High delay strongly indicates disengagement
Total Spend Numeric $100 – $1,000 🟡 Medium Higher spend = lower churn (sticky customers)
Tenure Numeric 1 – 60 months 🟡 Medium Short tenure customers churn more often
Usage Frequency Numeric 1 – 30 /month 🟡 Medium Low usage predicts disengagement and churn
Subscription Type Categorical Basic / Standard / Premium 🟡 Medium Premium subscribers are most loyal
Age Numeric 18 – 65 🟢 Low Minor influence on churn in this dataset
Last Interaction Numeric 1 – 30 days 🟢 Low Recency of last touchpoint
Gender Categorical Female / Male 🟢 Minimal Very low predictive power for churn
🚦 Risk Tier System
🔴 HIGH RISK
Churn Probability: 65% – 100%

• Assign dedicated CSM
• Offer contract upgrade discount
• Win-back campaign within 48hrs
🟡 MEDIUM RISK
Churn Probability: 35% – 65%
Proactive Outreach:
• Schedule a check-in call
• Send usage tips & highlights

🟢 LOW RISK
Churn Probability: 0% – 35%

• Collect NPS feedback
📂 Batch CSV Upload Guide

Required CSV Column Headers

Age, Gender, Tenure, Usage Frequency,
Support Calls, Payment Delay,
Subscription Type, Contract Length,
Total Spend, Last Interaction

ℹ️ CustomerID is optional and will be preserved in output but not used for prediction. Column order does not matter.

Valid Categorical Values

Column Valid Values
Gender Female | Male
Subscription Type Basic | Standard | Premium
Contract Length Annual | Monthly | Quarterly

⚠️ Values must match exactly (case-sensitive). Rows with missing values will be skipped.

🛠️ Troubleshooting
Problem Cause Fix
⚠️ "Model not found" error .joblib files missing Run python train_model.py first
⚠️ CSV upload fails Wrong column names or missing cols Check headers match the required list exactly
⚠️ LabelEncoder error Unknown category value in CSV Ensure Gender/Subscription/Contract values match valid options
⚠️ streamlit not found Package not installed Run pip install streamlit
⚠️ Model too slow Large batch or slow machine Reduce n_estimators in train_model.py to 50
